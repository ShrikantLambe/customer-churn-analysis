# Contributing Guidelines

Thank you for considering contributing to the Customer Churn Analysis project!

## How to Contribute
- Fork the repository and create your branch from `main`.
- If you've added code, ensure it is well tested.
- If you've changed APIs, update the documentation.
- Ensure your code passes linting and tests before submitting a PR.

## Project Structure
- `src/` — Core source code (preprocessing, training, evaluation, utils)
- `notebooks/` — Jupyter notebooks for EDA and prototyping
- `app/` — Streamlit app for interactive prediction
- `models/` — Saved model files (not versioned in git)
- `data/` — Raw and processed datasets (not versioned in git)
- `tests/` — Unit and integration tests

## Coding Standards
- Use Python 3.8+ and follow PEP8.
- Write docstrings for all public functions and classes.
- Use type hints where possible.

## Testing
- Add tests for new features in the `tests/` directory.
- Run all tests with `pytest` or `unittest` before submitting.

## Documentation
- Update the README.md and docstrings as needed.
- For major changes, update or add Jupyter notebooks in `notebooks/`.

## Issues
- Use GitHub Issues for bug reports and feature requests.

Thank you for helping make this project better!
